# 赛乐文档中心与赛乐助手配置说明

## 1. 创建知识库「赛乐文档中心」

### 方式一：在管理后台创建（推荐）

1. 登录管理后台，进入 **知识库**。
2. 点击 **新建知识库**。
3. 填写：
   - **名称**：赛乐文档中心
   - **描述**：赛乐文档中心知识库，用于赛乐助手检索。
   - **嵌入模型**：选择当前知识库配置中的默认嵌入模型。
4. 保存后，上传或导入所需文档。

### 方式二：通过 API 创建（需管理员登录态）

使用已登录管理员的 Cookie/Token 调用：

```bash
# 获取知识库默认嵌入模型 ID（需先配置 系统配置 -> 知识库模型）
# 然后创建知识库（将 <embedding_model_id> 替换为实际 ID）
curl -X POST "${BASE_URL}/api/v1/knowledge/create" \
  -H "Content-Type: application/json" \
  -H "Cookie: ..." \
  -d '{
    "name": "赛乐文档中心",
    "description": "赛乐文档中心知识库",
    "type": 0,
    "model": "<embedding_model_id>"
  }'
```

## 2. 创建工作流「赛乐助手」

在管理后台 **工作流** 中新建工作流，按以下设计配置。

### 功能要求

- **查询知识库**：使用「知识库检索」节点，关联上面创建的「赛乐文档中心」。
- **联网**：可增加「工具」节点选择 `web_search`（需在工作台或工具管理中配置好联网搜索），或由 LLM 在回答中建议用户使用工作台自带的联网搜索。
- **数据库数据**：若系统中有数据库查询类工具，可增加对应工具节点；否则在系统提示词中说明“涉及数据库数据请根据已有知识回答或建议联系管理员”。

### 回答规范（必加）

在 **LLM 节点** 的 **系统提示词** 中加入以下约束（可根据需要微调）：

```
你是一名合规的赛乐助手。请严格遵守以下规定：

1. 仅基于当前知识库检索结果、联网搜索结果（若启用）以及合规常识回答用户问题；不得编造、臆测或传播未经核实的信息。
2. 禁止回答或协助任何违反中华人民共和国法律法规、公序良俗、伦理道德的内容。
3. 禁止回答涉及暴力、违法、欺诈、侵权、危害国家安全或社会稳定的内容。
4. 若问题超出知识库与合规范围，或涉及敏感/违规主题，应礼貌拒绝并说明原因。
5. 涉及实时数据、数据库数据时，可建议用户通过工作台联网搜索或联系管理员获取；不得虚构数据。
6. 回答应简洁、准确、专业，并注明重要结论的来源（如“根据知识库文档”“根据检索结果”等）。
```

### 建议节点流程

1. **开始** → **输入**（对话输入，接收用户问题）
2. **输入** → **知识库检索**（选择「赛乐文档中心」，检索条数按需设置）
3. **知识库检索** → **LLM**（系统提示词使用上面的规范；用户提示词中引用知识库检索结果变量）
4. 若需联网：可增加 **工具** 节点（web_search），再接入 LLM；或由 LLM 提示用户使用工作台联网。
5. **LLM** → **结束**

创建并保存后，将工作流 **发布**（上线），并复制其 **工作流 ID**（在详情或 URL 中可见）。

---

## 3. 将工作台「宁伊助手」关联到赛乐助手工作流

1. 进入 **创建智能体 -> 工作台 -> 日常**。
2. 在 **宁伊助手关联工作流 ID** 中填入上一步复制的「赛乐助手」工作流 ID。
3. 保存配置。

保存后，用户在工作台侧边栏点击 **宁伊助手** 时，将自动打开「赛乐助手」工作流对话，而不再使用工作台默认对话。

---

## 4. 可选：通过脚本创建知识库并写回配置

若已配置好嵌入模型且希望用脚本创建知识库，可在项目根目录执行（需 Python 环境与 MEP 依赖，且需能连接后端/数据库）：

```bash
cd /path/to/MEP_Dev
# 创建知识库（需根据实际环境修改脚本中的 request/user 等）
python scripts/init_saile_knowledge.py
```

脚本会创建名为「赛乐文档中心」的知识库；工作流与关联配置仍需在管理后台按上述步骤完成。
